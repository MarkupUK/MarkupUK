XProc as an Application Engine

Erik Siegel

XProc as an Application Engine

This presentation explores the use of XProc 3.0 in developing a set of command-line applications for converting XML documents into various derivatives. The author uses XProc as the core of the application and experiments with building the application logic in XProc, including command-line handling and file/directory management. The paper proposes two design patterns that can be widely applied to use XProc effectively in developing command-line applications.
10:00 - 10:40

Working with XML Inside a Web Browser - Edge Computing for XML

Daniel Arthursson

Working with XML Inside a Web Browser - Edge Computing for XML

Decentralization of data and applications with edge computing is coming outside the XML community. Can we revitalize web browsers and shift XML processing from the server-side to the client-side with added benefits of improved speed and low-latency? Can we make XML an alternative to Javascript and JSON development within web browsers? This requires client-side database technologies, client-side XML programming languages, support for XSLT, and powerful ways to address standalone XML documents as well as XML stored in document databases or XML supplied through cloud-based API calls. This calls for data abstraction and a unified data model within the web browser. This paper talks about two large projects aimed at solving the client-side equation of XML and put XML on the map for web development again.

Enhancing Markup Quality Assurance with Automated Schema Visualization

Sven Reinck

Enhancing Markup Quality Assurance with Automated Schema Visualization

The paper describes an XSD visualizer plugin that automates schema visualization and emphasizes the accuracy and completeness of element structure. It provides a comprehensive view of the inheritance hierarchy and actual element structure, allowing users to quickly identify potential errors and ensure the correctness and completeness of markup documents. This tool is useful for academics, software engineers, and end-users interested in enhancing their markup quality assurance practices through automated schema visualization with a specific focus on element structure.

ISO Standards Update

Francis Cave

ISO Standards Update

ISO/IEC JTC 1 is the joint technical committee of ISO and IEC that is responsible for IT standardization. SC 34 is the sub-committee of JTC 1 that is concerned with the active maintenance of several widely-used document description and processing language standards, including the schema language Schematron (ISO/IEC 19759-3) and office file format standards OOXML (ISO/IEC 29500), ODF (ISO/IEC 26300 in collaboration with OASIS) and standards supporting the use of EPUB for e-books. The presentation will include a brief update on the current work programme of SC 34.

Improving Quality Critical XML Workflows with XProc 3.0 Pipelines

Achim Berndzen

Thorsten Rohm

Improving Quality Critical XML Workflows with XProc 3.0 Pipelines

The paper discusses the challenges faced in enhancing the quality of XML pipelines. The existing pipelines are based on batch scripts and XSLT. They produce high-quality XML results but lacked additional quality features. The paper describes the additional features that were added to enhance the pipeline quality, such as validating results, verifying links to image documents, and producing only valid transformation results. The paper discusses how XProc 3.0 was used to improve the pipeline, reduce dependencies and enable the creation of summarized logs.

Utilizing an XSLT/XProc-based Framework.

Martin Kraetke

Bridging the Gaps between XML and LaTeX. An Approach to Convert from Arbitrary XML to LaTeX by Utilizing an XSLT/XProc-based Framework.

There are established software solutions to convert from XML to TeX. Nevertheless, these approaches either limit the configurability of the TeX output or introduce a lot of programming effort. In this paper, we will discuss these methods and present an alternative approach that hopes to offer the greatest possible flexibility based on fixed conversions, an extensible configuration and the injection of XSLT, and that forms the basis for our OPen Source typesetting system xerif.

Return on Experience : Using TDD to Produce High Quality XSLT

Christophe Marchand

Return on Experience : Using TDD to Produce High Quality XSLT

This paper discusses the benefits of using Test-Driven Development (TDD) on XSLT development, specifically using the XSpec unit-test framework. Through TDD, the resulting XSLT code is of high quality with no bugs, easy to maintain, and well-documented by tests with 100% code coverage. The paper also includes a live-coding session demonstrating how to apply TDD and refactoring operations such as Extract Function.

Your Daddy's Word Processor Is So Last-Century

Kurt Conrad

Your Daddy's Word Processor Is So Last-century

This presentation describes Kurt's journey of replacing WordPerfect with prodoc.dtd, a semantic authoring doctype; and how prodoc evolved to enable computer-assisted sense making, based on markup that formalizes knowledge flow analysis and modelling semantics. The paper also describes some of the challenges associated with multi-perspective decision making and techniques for negotiating and formalizing dynamic, natural-system ontologies. Lessons learned from WordNet and SUMO integrations are summarized. The paper concludes by suggesting that building editors and bots that use author-authored markup to digitize and amplify logic systems could, in many organizational settings, contribute to more-intelligent value optimizations and better long-term performance.

Linguistic Quality Assurance

Lingunixtix

Linguistic Quality Assurance

This presentation discusses how markup languages have their own vocabularies, dialects, and grammars, and can be translated into and from other languages and formats. The author suggests that instead of creating ad-hoc linter rules, lessons can be drawn from the tree of linguistics. The presentation covers topics such as the history of semantic markup, quantitative aspects of document processing, metrics for measuring readability and complexity, citation styles, the benefits of corpora and translation memories, and new editor macros and techniques inspired by Chomsky's hierarchy.

Leveraging the Power of OpenAI and Schematron for Content Verification and Correction

Octavian Nadolu

Leveraging the Power of OpenAI and Schematron for Content Verification and Correction

This presentation aims to give an overview of AI and its potential benefits for content verification and correction using Schematron and SQF. OpenAI, a research lab focused on AI, has developed algorithms that can be used with Schematron to automate the verification of content. This can help save time and money for content creators and publishers while improving content accuracy. However, there are challenges in using OpenAI algorithms, such as the possibility of inaccurately identifying errors and not being able to detect cultural or regional differences in language. It is important to consider these challenges when using OpenAI for content verification and correction.

XQS: A Native XQuery Schematron Implementation

Andrew Sales

XQS: A Native XQuery Schematron Implementation

This presentation will cover XQuery for Schematron (XQS), a Schematron processor being implemented in native - and naive - XQuery. To the author's knowledge, there are no other such implementations available at this time. The purpose of the work is at least two-fold: to demonstrate the utility of an XQuery query language binding for arguably the primary quality assurance technology applicable to XML, and to provide a second "reference" implementation while work on the latest revision of the ISO standard proceeds. (There is no formal requirement for any implementation under ISO regulations, but it clearly helps to be able to answer the question standards authors should consider when new features present themselves: How would you implement that?)

XSLT and XQuery 4.0: Work in Progress

Michael H Kay

XSLT and XQuery 4.0: Work in Progress

Back in 2020, the author outlined some ideas for new features to go into a proposed XSLT 4.0. This talk gives an update on how those ideas are progressing. A W3C Community Group has been formed to take proposals forward and is meeting weekly; the current state of play is visible at qt4cg.org for those who are interested.

Inevitably, there's a vast amount of detailed change, and standing back to examine what's significant, what the overall themes are, isn't all that easy. Perhaps the best way to summarize is to say that we're adding functionality and convenience, all building on the solid foundations of the 3.0/3.1 specifications. In 3.1 we introduced major new infrastructure: arrays, maps, higher order functions; but there's a feeling that it's not fully exploited. For 4.0, the aim is to leave the foundations intact, and to build on them. The talk will aim to have both breadth and depth: giving a lightning tour of the range of new features in the language(s), and selecting one or two areas for a deeper examination.

A Dependency Management Approach for Document and Data Transformation Projects

Jorge Sanchez

A Dependency Management Approach for Document and Data Transformation Projects

This talk introduces and evaluates the suitability of Apache Ivy as a dependency manager for document and data transformation projects, which is less common in these projects. The proposed solution will provide an optimized set of templates based on the development of Arousa and test common scenarios like dealing with local repositories and setting up a shared network repository. The approach has shown that it is suitable for dependency management on XSLT and XProc transformation projects, and the code samples will be shared for future reference. The initial experience with Arousa seems to back the validity of the solution, and the templates provided can help with the added complexity.
